Projet : LIBRAIRIE - Gestion de Livres

← Retour aux projets

Aperçu du projet

Description du projet

Ce projet est une application web simple de gestion de livres, permettant aux utilisateurs de s'inscrire, de se connecter, et de gérer une liste de leurs livres favoris. L'application est développée en PHP avec une base de données MySQL et utilise PDO pour les interactions avec la base de données. pensé pour un déploiement local

Fonctionnalités

  • Inscription des utilisateurs : Création de comptes avec nom d'utilisateur, e-mail et mot de passe sécurisé.
  • Connexion des utilisateurs : Accès sécurisé aux comptes enregistrés.
  • Gestion des favoris : Consultation et suppression de livres de la liste personnelle.
  • Navigation : Barre de navigation pour accéder aux différentes sections de l'application.
  • Interface Utilisateur : Design simple, moderne et réactif avec CSS.

Technologies utilisées

PHP MySQL/MariaDB PDO HTML5 CSS3

Structure du Code

Le projet est organisé avec des fichiers dédiés pour la gestion de la base de données, l'authentification des utilisateurs et les différentes pages de l'application :

  • database.php : Gère la connexion à la base de données MySQL en utilisant PDO.
  • user.php : Contient la classe `User` pour la logique d'authentification.
  • inscription.php : Gère le processus d'inscription des nouveaux utilisateurs.
  • mes_favoris.php : Affiche et gère la liste des livres favoris de l'utilisateur.
  • style.css : Fichier de style CSS pour l'ensemble de l'application.
  • Autres fichiers (index.php, accueil.php, profil.php, logout.php) : Gèrent les pages principales de l'application.

Ce projet a permis de renforcer la maîtrise du développement web côté serveur avec PHP et MySQL, la gestion des sessions utilisateurs et l'interaction avec une base de données via PDO.

Résolution d'incident : Fichier de configuration manquant

Durant le développement, un incident critique a empêché le lancement de l'application. Voici un résumé simple du diagnostic et de la résolution.

❗️

Problème

L'application était inaccessible à cause d'une erreur PHP fatale indiquant que le fichier config.php était introuvable.

🔍

Cause

Le fichier config.php, contenant la clé API, avait été placé dans un mauvais dossier lors d'une réorganisation du projet.

✅

Solution

La résolution a consisté à déplacer le fichier config.php à la racine du projet. L'application est redevenue fonctionnelle instantanément. Ce fichier est volontairement exclu de GitHub via le .gitignore pour protéger ma clé API, garantissant ainsi la sécurité des données sensibles du projet tout en assurant son bon fonctionnement en local.
